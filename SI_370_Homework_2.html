In[]:
def decade(year):
    
    if year.year < 1910:
        return '1900s'
    if 1910 < year.year < 1920:
        return '1910s'
    if 1920 < year.year < 1930:
        return '1920s'
    if 1930< year.year < 1940:
        return '1930s'
    if 1940< year.year < 1950:
        return '1940s'
    if 1950< year.year < 1960:
        return '1950s'
    if 1960< year.year < 1970:
        return '1960s'
    if 1970< year.year < 1980:
        return '1970s'
    if 1980< year.year < 1990:
        return '1980s'
    if 1990< year.year < 2000:
        return '1990s'
    if 2000< year.year < 2010:
        return '2000s'
    if 2010< year.year < 2020:
        return '2010s'
In[]:
def good_or_bad(rating):
    if rating < 3.5:
        return 'bad'
    if rating >= 3.5:
        return 'good'
In[]:
mean_movie_ratings = ratings.groupby('movieId')['rating'].mean().reset_index()
mean_movie_ratings['quality'] = mean_movie_ratings.rating.apply(good_or_bad)
movies_with_mean_rating = movies.merge(mean_movie_ratings, on='movieId', how='inner')
movies_with_mean_rating.head()
movies_with_mean_rating['decade_released']= movies_with_mean_rating['year'].apply(decade)
rating_counts = ratings.groupby('movieId').size().reset_index(name='number_of_ratings')
df = movies_with_mean_rating.merge(rating_counts, on='movieId', how='inner')

This graph is somewhat hard to interpret due to the the fact that the datapoints are so small, and I am also not super familiar with violin plots. The 1990s movies have the most good ratings, but since a violin plot shows kernel density, the rating count doesn't matter as much when interpretting the plot. Skinnier sections of a violin plot mean that there is a lower probability that the ratings will fall around that number. Wider sections of the violins represent higher probability that data in that group will take on that rating.
